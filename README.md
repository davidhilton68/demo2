  Root Cause Still Under Investigation — getUser("admin") Returns Null                                                                                                                                                                                                                     
                                                                                                                                                                                                                                                                                           
  Despite all of the above, login still fails with UsernameNotFoundException: Unknown username : admin.
                                                                                                                                                                                                                                                                                           
  Root cause identified (but not yet fixed):                                                                                                                                                                                                                                             
                                                                                                                                                                                                                                                                                           
  IkasanSecurityConfiguration (in the dashboard's bean configuration) defines:                                                                                                                                                                                                             
   
  @Bean(name = "entityManagerFactory")                                                                                                                                                                                                                                                     
  public LocalContainerEntityManagerFactoryBean configurationServiceEntityManager(...) {                                                                                                                                                                                                 
      ...                                                                                                                                                                                                                                                                                  
      localContainerEntityManagerFactoryBean.setPersistenceUnitName("configuration-service");
      localContainerEntityManagerFactoryBean.setPersistenceXmlLocation("classpath:/configuration-service-persistence.xml");                                                                                                                                                                
      ...                                                                                                                                                                                                                                                                                  
  }                                                                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                                           
  This overrides IkasanWebAutoConfiguration.entityManagerFactory() which was intended to be for the "security" persistence unit. The result is that the Spring context has no EntityManagerFactory registered for the "security" persistence unit.                                         
   
  When HibernateUserDaoImpl (and HibernateSecurityDaoImpl) use @PersistenceContext(unitName = "security"), Spring's PersistenceAnnotationBeanPostProcessor cannot find an EntityManagerFactory with persistence unit name "security" and falls back to the default "entityManagerFactory"  
  bean — which is for "configuration-service", not security entities.                                                                                                                                                                                                                    
                                                                                                                                                                                                                                                                                           
  As a result:                                                                                                                                                                                                                                                                             
  - HibernateUserDaoImpl.getUser("admin") returns an empty result (no SQL logged for the Users table)
  - HibernateSecurityDaoImpl.getAuthenticationMethods() returns an empty list                                                                                                                                                                                                              
  - Login fails with UsernameNotFoundException                                                                                                                                                                                                                                           
                                                                                                                                                                                                                                                                                           
  The admin user does exist in H2 with the correct bcrypt password and enabled status — this is confirmed by direct H2 shell query.                                                                                                                                                      
                                                                                                                                                                                                                                                                                           
  Suggested fix for developers: IkasanSecurityConfiguration.configurationServiceEntityManager() should not use @Bean(name = "entityManagerFactory"). It should use a unique name (e.g., @Bean(name = "configurationServiceEntityManagerFactory")), and a separate, properly named bean     
  should be created for the "security" persistence unit — or SecurityAutoConfiguration should be properly registered as a Spring Boot auto-configuration so that its securityEntityManager bean is loaded.
